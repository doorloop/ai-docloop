# Examples

| File                                       | Flow                                                                                                                                            |
| ------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| [`single-step.yml`](./single-step.yml)     | Minimum viable workflow — one step, one mapping, defaults everywhere.                                                                           |
| [`wiki-insights.yml`](./wiki-insights.yml) | Multi-step workflow that maintains both per-feature insight pages (delivered as a PR) and per-component READMEs (delivered as a direct commit). |

Copy either file into `.github/workflows/` in the consumer repo. See the project [README](../README.md) for the full input list and behavior reference.
